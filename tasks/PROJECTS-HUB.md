## Как работает флоу

### Ты (телефон, голос)
1. Открываешь Claude Chat на телефоне
2. Наговариваешь всё что в голове — мысли, задачи, идеи, проблемы
3. Просишь Claude: **"Структурируй для GitHub issues"** или **"Обнови мой хаб"**
4. Получаешь готовые задачи — копируешь в issue или скидываешь сюда

### Claude Code (исполнение)
1. Читает этот файл — знает приоритеты
2. Берёт задачу: "Возьми TASK-008" или "Сделай афишу для Edison"
3. Ruflo роутит по агентам если нужно
4. Делает → коммитит → пушит → отчёт

### Голосовой шаблон для Claude Chat
```
Привет, я Frogface. У меня обновление по проектам:

[Наговори что хочешь — потоком, без структуры]

Пожалуйста:
1. Выдели задачи и раздели по проектам (MyReply / Edison / Frogface)
2. Расставь приоритеты (critical / high / medium / low)
3. Оформи каждую задачу в формате:

**Проект:** название
**Задача:** что сделать
**Приоритет:** critical/high/medium/low
**Детали:** контекст

4. Если есть что обновить в хабе — скажи что поменять
```

## Формат задач

Файлы в `tasks/{project}/TASK-{NNN}-{slug}.md`:

```markdown
# TASK-NNN: Название

**Статус:** pending | in_progress | done
**Приоритет:** critical | high | medium | low
**Проект:** myreply | edison | frogface
**Создано:** YYYY-MM-DD
**XP:** число

## Описание
Что нужно сделать

## Критерии готовности
- [ ] Пункт 1
- [ ] Пункт 2

## Результат
(заполняется после выполнения)
```

## Быстрые команды для Claude Code

| Что сказать | Что произойдёт |
|---|---|
| "Покажи все задачи" | Сканирует tasks/, выводит список |
| "Возьми TASK-008" | Читает задачу, выполняет, обновляет статус |
| "Новая задача для MyReply: ..." | Создаёт TASK файл |
| "Что горит?" | Показывает critical задачи |
| "Сделай афишу для Edison" | canvas-design + текст |
| "Обнови хаб" | Обновляет этот файл |
