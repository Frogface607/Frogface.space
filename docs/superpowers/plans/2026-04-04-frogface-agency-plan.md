# Frogface Agency — Implementation Plan

**Spec:** `docs/superpowers/specs/2026-04-04-frogface-agency-design.md`
**Date:** 2026-04-04

## Phase 1: Foundation (Steps 1-3)

### Step 1: Supabase Schema + Seed Data
**Files:** `supabase/agency-schema.sql`
**Action:** Apply migration via Supabase MCP
- Tables: `agents`, `departments`, `conversations`, `agent_messages`, `artifacts`, `knowledge_base`, `hiring_pool`
- Seed: 4 frogs (Scribe, Kvax, Sage, Lilypad) with full personality prompts
- Seed: default channels (#general, #myreply, #edison, #frogface, #meeting)
- Seed: hiring pool (Analyst, Designer, Content Creator, Sales, Guardian)
- **Verify:** Query tables to confirm structure and seed data

### Step 2: Agency API Routes
**Files:** Create in `src/app/api/agency/`
- `chat/route.ts` — POST: send message, trigger Claude API for frog response
  - Loads frog system_prompt + conversation history + knowledge_base context
  - Returns frog response, saves both messages to DB
- `agents/route.ts` — GET: list all frogs with status/mood
- `conversations/route.ts` — GET: list channels and DMs
- `messages/[id]/route.ts` — GET: messages for a conversation (paginated)
- `knowledge/route.ts` — GET: search KB, POST: add entry
- **Key:** Chat route must include autonomy level logic (autopilot vs tinder-approve vs conversation)
- **Verify:** curl/fetch each endpoint, confirm responses

### Step 3: Agency Page — Layout Shell
**Files:** `src/app/(dashboard)/agency/page.tsx`, `src/app/(dashboard)/agency/layout.tsx`
- Rename sidebar `/studio` link to `/agency`
- Top zone: simple frog status cards (name, avatar emoji, status, current task) — NOT full map yet
- Bottom zone: channel list (left) + chat area (right)
- Mobile: channel list as horizontal tabs, full-screen chat
- Wire to API: load agents, conversations, messages on mount
- **Verify:** Page renders, shows frogs and channels from Supabase

## Phase 2: Chat System (Steps 4-5)

### Step 4: Single Frog Chat
**Files:** Modify `src/app/api/agency/chat/route.ts`, agency page components
- Text input + send button
- Messages display with frog avatar, name, timestamp
- Architect messages with crown icon
- Typing indicator while waiting for Claude API
- Conversation history stored in Supabase (not localStorage)
- Each frog uses personalized system prompt from DB
- **Verify:** Send message to Kvax, get response, see it in chat

### Step 5: Multi-Frog Meetings
**Files:** `src/app/api/agency/meeting/route.ts`
- POST `/api/agency/meeting` with topic
- Creates meeting conversation, selects relevant frogs by topic keywords
- Round-robin: each frog responds seeing all prior messages
- Meeting ends with summary artifact (auto-generated by Scribe)
- UI: meeting messages tagged with participant frog avatars
- **Verify:** Start meeting about "Edison marketing", see 3+ frogs discuss

## Tech Notes

- Claude API via Anthropic SDK (not OpenRouter) — direct, reliable
- System prompts stored in DB (agents table) — editable without deploy
- All frog personalities in Russian, meme-style
- Messages paginated (50 per load) to keep UI fast
- Supabase realtime subscriptions for live message updates (future)
