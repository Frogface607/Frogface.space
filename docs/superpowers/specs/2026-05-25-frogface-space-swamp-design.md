## 0. TL;DR

**frogface.space = живое 2.5D болото в WIZL-стиле, по которому ходит лягуха-Серж (маскот = визуализация Босса), заходит в здания-проекты, и мир растёт по мере новых кейсов и роста Босса.**

Сайт не статичная витрина и не классический портфолио. Это **интерактивный визуальный дневник пути из болота к небоскрёбу**, нарисованный в эстетике Studio Ghibli × Moomin × Carson Ellis × Phoebe Wahl. Контент-фабрика стоит на **Nano Banana Pro (безлимит на Freepik Premium) + Seedance 2 (image-to-video) + опционально Higgsfield Soul-ID** для консистентности персонажа.

## 1. Ядро — метафора пути

### Сюжетная арка

```
БОЛОТО → БЕРЕГ → ДЕРЕВНЯ → ГОРОД → НЕБОСКРЁБ
(где Босс сейчас)  (где будет)              (когда богатый и знаменитый)
```

Сайт стартует в режиме **БОЛОТО** (`wealth_tier = "swamp"`). Tier-флаги Босс **флипает руками** когда его жизнь/доходы прокачиваются (это его ритуал, а не автоматика). Сцена плавно эволюционирует: появляются деревянные мостки, потом каменная улица, потом стеклянный небоскрёб, к которому ведут все старые тропы. Болото никогда не исчезает — оно остаётся внизу кадра, виднеется через окно небоскрёба, чтобы помнить «откуда».

### Что считает посетитель за 5 секунд

> «Этот чувак в виде лягухи живёт в болоте, делает интересные штуки, и у него растёт свой мир. Хочу так же. Дай посмотреть его проекты.»

Это **встроенный продающий нарратив** для Studio без единого слова «продаём».

### Маскот = визуализация Босса (важно)

Лягуха-Серж — это **Босс в образе**. Не «персонаж с моим характером». Буквально: «вот я в виде лягухи, я сейчас в болоте, я тут работаю, у меня лавовая лампа, кружка кофе, ноутбук». Без сложного лора, без супергеройских масок, без магии — просто **визуальное альтер-эго**, которое позволяет Боссу не светить лицом, но присутствовать.

## 2. Визуальный язык — WIZL-DNA

### Эстетика

**Cozy psychedelic illustrated world.** Не bold comic, не нуар, не Pixar, не аниме.

Опорные референсы (с WIZL-канона):
- **Studio Ghibli** (особенно Spirited Away, My Neighbor Totoro) — painterly watercolor, мягкие формы, живой мир
- **Moomin** (Tove Jansson) — уют, странность, дружелюбные «существа»
- **Carson Ellis × Phoebe Wahl** — иллюстрированные книжки для взрослых, природа, folk-magic
- **Alex Face** — стрит-арт Bangkok, выразительные глаза, простые формы
- **Moebius** + **Amano** — для отдельных «арт-объект» вставок (постеры проектов внутри оверлеев)

### Что НЕ берём

- ❌ Bold modern comic с ink-outlines (отменили после анализа WIZL-канона)
- ❌ Marvel-стилистика, ink-контуры толстые
- ❌ Pixar, anime, hyper-realism
- ❌ Дарк-нуар, графит-вампирский (отменили v1)
- ❌ Кислотные неоны
- ❌ Эпик-магия (Босс: «без особой магии»)

### Палитра проекта (CSS-токены, обновлены по WIZL-канону)

```css
/* основа */
--swamp-deep:     #2a2640    /* deep mystic purple, основной фон ночи */

--swamp-mist:    #6b7d6e    /* туман / горизонт */
--paper-cream:    #f2e8d4    /* WIZL bright paper, тексты на тёмном */
--paper-warm:     #e8d8a8    /* тёплая бумага, оверлеи */

/* акценты — спарки, действия, свет */
--gold-warm:      #e9c46a    /* тёплые окна, вывески, gold leaf */
--lime-spark:     #99f788    /* светлячки, "хочу гулять" подсветка, WIZL signature */
--stardust-blue:  #7dc4ff    /* блёстки магии, ночное небо */
--ember-pink:     #ff6b9d    /* редкий акцент эмоции, sunset */
--ink-soft:       #1a1320    /* мягкий "чёрный" с фиолетовым подтоном */

/* WIZL bright marks */
```

### Типографика

- **Display / заголовки**: hand-lettered, slightly imperfect — например **Cinzel Decorative** или **Carattere** (как master lockup WIZL). С crystal-glow inner highlight. Никаких Bangers / Comic Sans.
- **Body**: тёплый sans с характером — **Space Grotesk** уже стоит, оставляем но снижаем веса до 400/500. Или альтернатива — **DM Sans**, **Fraunces** для душевности.
- **Mono / лейблы**: **JetBrains Mono** (уже подключён) — для timestamps, координат, мета-инфо.
- **Special**: для арт-объект-вставок внутри оверлеев — пер-эпизод-стиль типографики из WIZL-канона (frost, ice, brushwritten, etc).

### Освещение и текстуры

- **Watercolor wash** подложка на всех больших картинках
- **Soft glow** на источниках света (окна, фонари, светлячки) — gold-warm + lime-spark
- **Mist particles** — лёгкий туман плывущий справа-налево
- **Film grain** очень лёгкий поверх сцены — добавляет тёплости, не агрессивно
- **No hard ink outlines** — формы определяются цветом и тенью, не контуром

## 3. Главный персонаж

### Лягуха-Серж — визуализация Босса

Антропоморфная лягуха, мягкий «дымчатый» силуэт (без жёстких контуров), уютный, добрый, с характерными узнаваемыми чертами. **Не похожа на Босса лицом** (Босс не хочет светить), но **визуализирует его внутреннее состояние**.

**Внешний вид:**
- Скин: мягкий зелёный с тёплым оттенком (не яркий зелёный, скорее «мшистый»)
- Глаза: большие, выразительные, чуть уставшие — Alex Face × Moomin реверанс
- Одежда: oversized тёплый худи (тёмно-фиолетовый или болотный графит), широкие штаны, простые кеды
- Силуэт: округлый, не мускулистый, не карикатурный — «обычный чел в виде лягухи»

**Зачем без жёстких контуров.** Это и есть WIZL-DNA. Жёсткий комикс-outline = Marvel = «корпорат». Soft painterly = Ghibli = «уютно, своё». Босс уже принял этот стиль для WIZL, для frogface держим консистентно.

### Гибрид: гуляющий + работающие Сержи

Чтобы примирить **«играбельно» + «маскот сидит работает»**:

1. **Гуляющий Серж** — главный игровой персонаж. Им управляет посетитель (tap-to-walk / WASD). Он ходит по болоту, может подойти к любому зданию.

2. **Работающие Сержи** — в каждом проекте сидит **отдельный Серж**, работающий именно над этим проектом. Soul-ID locked (это **тот же Серж**), но в разных позах и контекстах:
   - У Edison-бара: стоит за стойкой, протирает стакан, тёплое окно
   - На WIZL-кочке: сидит с бонгом и книгой strain-ов, рядом светящиеся грибы
   - У MyReply-сарая: за ноутом с лавовой лампой, морда уставшая, рядом стопка отзывов
   - У Posadyat-стойки: с газетой в руках, картой РФ в спине
   - У Studio-вывески: с молотком и чертежами, конструирует что-то

3. **Встреча с собой.** Когда Гуляющий Серж подходит к Работающему — они «встречаются». Спич-баблы, лёгкая анимация (приветствие, взгляд). Это и есть приглашение «зайти внутрь» проекта.

Этот паттерн делает мир **по-настоящему живым** и решает противоречие «ходит / сидит». Босс одновременно и гуляет (как гость своего пространства), и работает (как мастер каждого проекта).

## 4. Технологическая архитектура

### Stack

| Слой | Решение | Зачем |
|---|---|---|
| Framework | Next.js 16 App Router + React 19 + TS | Уже стоит, остаёмся |
| Renderer (мир) | **PixiJS v8** | 2.5D-сцена, спрайты, parallax, GPU-ускорение, mobile-friendly |
| Анимация UI / оверлеи | Framer Motion 11 | Уже стоит |
| Smooth scroll (на статичных страницах) | Lenis | Уже стоит |
| Particles | PIXI ParticleContainer + кастомный update | Светлячки, туман, пыльцы — производительно |
| 3D на будущее | Three.js / R3F **опционально для tier-3+ комнат** | Только если когда-то Studio станет 3D-комнатой |
| MDX для контента проектов | next-mdx-remote | Тексты проектов в .mdx, удобно править |
| Конфиг мира | `worldState.json` в репо | Tier, здания, координаты — флипай руками |

### Content generation toolchain (главные правки vs v1)

| Step | Tool | Зачем |
|---|---|---|
| **1. Character base** | **Nano Banana Pro** (через nano-banana skill / Freepik Premium безлимит) | Главный image gen для статичных сцен и маскота. Базовый промпт → reference image |
| **2. Character consistency** | **Higgsfield Soul-ID** (опционально, если Nano Banana хромает на консистентности) | Lock главного персонажа, чтобы в каждой сцене Серж узнаваем |
| **3. Character sheets** | Nano Banana Pro + ручная сборка в Procreate/Figma | Лист поз: idle, work-with-lamp, walk-1..6, sit, wave, drink-coffee, sleep |
| **4. Backgrounds & buildings** | Nano Banana Pro | Слои фона болота, здания, объекты |
| **5. Animation (loops)** | **Seedance 2** (Freepik Premium, image-to-video, до 10s) | Микро-лупы: моргает, дышит, листает книгу, поднимает кружку. 2-4 сек .mp4 |
| **6. Sprite sheets** | TexturePacker / ручная сборка | Финальные ассеты для PixiJS |

**Higgsfield Plus план Босса** — остаётся в резерве для специальных кадров (видео с реальным движением героя, кинематографичные шоты для постеров).

### Слойный пирог сцены

```
z-index 10 : UI overlay (HUD-индикатор tier, спич-баблы, hint-курсор)
z-index  9 : Particles (светлячки, мошки, пыльцы)
z-index  8 : Гуляющий Серж (sprite, реагирует на input)
z-index  7 : Foreground (трава, тростник, frontmost кочки) — parallax x1.3
z-index  6 : Interactive objects (бар, кочка-WIZL, домики + Работающие Сержи) — parallax x1.0
z-index  5 : Mid-ground (болото-средний слой) — parallax x0.8
z-index  4 : Background (дальний горизонт, дальние силуэты) — parallax x0.4
z-index  3 : Sky/atmosphere (небо, луна, drift облака) — parallax x0.1
z-index  2 : Fixed gradient sky base (deep purple → twilight)
```

### Управление (закрытые ответы Босса)

- **Desktop**: WASD / стрелки для ходьбы. Курсор-лягуха-морда (текущий компонент перерисовать под WIZL-soft-style).
- **Mobile**: **tap-to-walk** — тап в левую/правую часть экрана → лягуха идёт туда. Тап на здание (когда близко) → zoom-in оверлей.
- Никаких виртуальных джойстиков.

### Routing

- `/` — болото (PixiJS сцена, главный мир)
- `/projects/[slug]` — оверлей-страница проекта (Next.js intercepted routes — мир под оверлеем не размонтируется, URL deeplink-able)
- `/about` — **скролл-комикс биографии** (отдельный роут, не встроен в мир — Босс выбрал это в open question 4)
- `/now`, `/studio`, `/playground` — постепенно встраиваем в мир как локации, пока заглушки в общей эстетике

## 6. Контент-фабрика — Nano Banana Pro + Seedance 2

### Этап 0 — Character base + Soul-ID lock (1 вечер)

1. Готовлю **финальный character-promтext** для Босса в `prompts-nano-banana.md` (см. Раздел 9)
2. Босс генерит 6-8 черновиков через Nano Banana Pro
3. Выбираем эталон
4. **Опционально** — прогоняем через Higgsfield Soul-ID для лока (если консистентность хромает в дальнейших сценах)

### Этап 1 — Стартовая библиотека ассетов (3-4 вечера)

| Asset | Кол-во | Tool |
|---|---|---|
| Фон болота, 5 separately-layered PNG | 5 | Nano Banana Pro |
| Гуляющий Серж — 8 поз walk-cycle | 8 | Nano Banana Pro |
| Гуляющий Серж — эмоции (моргает, машет, чешется, прыжок) | 4 | Nano Banana Pro |
| Работающий Серж × 6 проектов | 6 | Nano Banana Pro |
| Здания/объекты (5-6 шт) | 5-6 | Nano Banana Pro |
| Облака drift PNG seq | 3-4 var | Nano Banana Pro |
| Грибы светящиеся, кочки, тростник, пеньки | 8-10 | Nano Banana Pro |
| Idle-животные второго плана (другие лягухи, насекомые) | 4-6 | Nano Banana Pro |

**Всё в одной WIZL-палитре, painterly Ghibli style.**

### Этап 2 — Seedance 2 анимации (1 неделя)

Каждый ключевой Серж получает 2-4 sec видео-луп для живости:
- Гуляющий Серж idle: дышит, моргает, иногда чешется
- Работающий Серж Edison: протирает стакан
- Работающий Серж WIZL: листает книгу, выпускает облако дыма
- Работающий Серж MyReply: печатает на ноуте, лавовая лампа пульсирует
- И т.д.

Лупы конвертируются в **PNG sprite sequences** для PixiJS (через ffmpeg / руками), либо проигрываются как `<video>` поверх сцены с прозрачностью где можно.

**Бонус**: все эти лупы переиспользуются в TG @sergeyorlove, IG, Reels. Одна работа = много каналов.

### Этап 3 — постоянка

- Каждую неделю — 3-5 новых сцен Сержа под текущие события / новый пост в TG
- Каждый новый проект = 1-2 generation для здания + 1 generation для Работающего Сержа
- Все ассеты живут в `public/world/` репо и в Obsidian-vault для переиспользования

### Workflow на каждый ассет

```
1. Промпт из prompts-nano-banana.md
   ↓
2. Nano Banana Pro генерит 4 варианта
   ↓
3. Босс выбирает 1
   ↓
4. Опц: Higgsfield Soul-ID правит консистентность персонажа
   ↓
5. Procreate / Photopea: ручной cleanup, прозрачный фон, разрезка слоёв
   ↓
6. Опц: Seedance 2 для image-to-video луп
   ↓
7. Сохраняем в /public/world/[asset-name].png (или .webp / .mp4)
   ↓
8. Регистрируем в worldState.json
   ↓
9. PixiJS подхватывает
```

## 7. Что мы НЕ делаем сейчас

- ❌ Полное 3D (Three.js/R3F). Отложили до Tier 3+ если вообще нужно. Сейчас 2.5D хватает per Босс.
